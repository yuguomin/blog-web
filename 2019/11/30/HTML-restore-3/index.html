HTML重塑(3)——链接和替换标签

Posted on 2019-11-30 | In HTML Restore   |   字数统计: 4.4k(字)   |   阅读时长: 16(分) | Visitors

前言

  • 本篇内容主要是HTML的功能部分学习，HTML是由标签承载的，功能的学习也就是标签的学习。以下是标签分类的链接和替换标签相关内容。

上一章总结了义和元信息标签，本章继续学习链接标签与替换标签。

链接标签

在第一篇中，我们讲到了HTML是超文本标记语言，超文本的特性就是带有超链接的文本，如果没有超链接，只能说是富文本。链接标签在HTML中就是对其他文档或者资源关系的实现，在HTML中，可以分为两种类型，一种是超链接型标签，一种是外部资源链接。这个分类的简单理解，可以说超链接型是被动链接，需要用户主动触发才会去跳转或者下载，而外部资源型链接是会主动下载的。

下面按照分类整理一下每个链接的使用方式。

link标签

link标签实际上也是元信息标签的一种，但它的使用都是靠链接进行的，并且在有些场景下可以把标签放在body中使用。所以我们在链接这一篇进行学习。
link标签生成的链接根据用处分类不同，可以分为超链接和外部资源链接。link标签的使用方式主要是用rel属性来区分，比如：

超链接型的link

超链接型的link标签就是被动型链接，在用户不操作的情况下，它们不会被主动下载。多是用来描述生成地址与当前页面的关系，都是元信息标签。

  1. canonical型link

这个标签用于标识当前页面的主url，当网站有多个url指向同一个页面的时候，搜索引擎访问这类页面时会去掉重复的页面，这个link会提示搜索引擎保留哪一个url。

  1. alternate型link

这个标签提示页面它的变形形式，这个所谓的变形可能是当前页面内容的不同格式、不同语言或者为不同的设备设计的版本，这种 link 通常也是提供给搜索引擎来使用的。

  1. prev和next型link
    在互联网应用中，很多网页都属于一个序列，比如分页浏览的场景，或者图片展示的场景，每个网页是序列中的一个项。这种时候，通过prev和next型link标签告诉搜索引擎或者浏览器它的前一项和后一项，这有助于页面的批量展示。

因为next型link告诉浏览器“这是很可能访问的下一个页面”，HTML标准还建议对next型link做预处理，这个具体需要看各个浏览器的实现。

  1. 其他超链接类的link
    其它超链接类 link 标签都表示一个跟当前文档相关联的信息，可以把这样的 link 标签视为一种带链接功能的 meta 标签。
  • rel="author"链接到本页面的作者信息或者联系方式，一般是 mailto: 邮箱地址或其他联系页面。
  • rel="help"链接到本页面的帮助页。
  • rel="license"链接到本页面的版权信息页。
  • rel="search"链接到本页面的搜索页面（一般是站内提供搜索时使用）。

接下来看一下外部资源型的link标签。

外部资源型的link

这一类的标签特点是会主动下载，并且根据 rel 类型做不同的处理。

  1. icon型link
    这个标签的作用体现在打开页面在浏览器标签上显示的小图标。它是唯一一个外部资源型的元信息link，其链接会被浏览器下载并用作网页icon，如果没有设定这样的link，多数浏览器会使用域名根目录下的favicon.ico。

HTML标准允许一个页面出现多个icon型link，并且用sizes指定它适合的icon尺寸。

  1. 预处理类link
    预处理的概念就是提前处理，我们知道打开一个页面一般会经过dns查询、建立链接、传输数据、加载进内存和渲染等一系列步骤。

预处理类link就是根据不同的程度，去控制浏览器提前针对一些资源完成一部分操作。主要有以下几种类型的link和功能：

  • dns-prefetch型link提前对一个域名做dns查询，这样的link里面的href实际上只有域名有意义。
  • preconnect型link提前对一个服务器建立tcp连接。
  • prefetch型link提前取href指定的url的内容。
  • preload型link提前加载href指定的url。
  • prerender型link提前渲染href指定的url。

主要的预处理控制的程度就是这些，但是具体的实现与否和细节处理还要看每个浏览器的实现。

a标签

a标签是anchor的缩写，也就是锚点的意思，对应到HTML中的作用就是标识一个特定的位置。a标签还有一个功能是链接其它目标。

具有 href 属性的a标签跟link一样，会产生一个超链接，用户不操作，不会进行下载，当点击时，会跳转到对应锚点或者地址。
具有 name 属性的a标签会当作一个锚点，用作跳转的目标，和 id 属性有一样的作用和唯一要求。

a标签也有rel属性，首先是和link相同的一些rel：

  • alternate
  • author
  • help
  • license
  • next
  • prev
  • search
    和link语义是相同的，但是link是不会被渲染的元信息标签，而a标签会被渲染在页面中。除了这几个还有一些其它rel属性值：
  • tag，表示本网页所属的标签；
  • bookmark，到上级章节的链接。比如博客集合。
    还有一些用于提示浏览器或者搜索引擎做一些处理的rel属性值：
  • nofollow，此链接不会被搜索引擎索引；主要是被一些使用人气排名技术的搜索引擎所使用。
  • noopener，此链接打开的网页无法使用opener来获得当前页面的窗口；当打开不受信任的链接时，这特别有用。
  • noreferrer，此链接打开的网页无法使用referrer来获得当前页面的url；
  • opener，打开的网页可以使用window.opener来访问当前页面的window对象，这是a标签的默认行为。主要是用于清除a标签target属性为”_blank”时隐式设置的rel="noopener"。

a标签基本解决了我们在页面中插入文字或者图片超链接的需求，如果我们想要在图片中插入超链接，并且支持非矩形点击区域，那么就需要使用area标签了。

area标签

area标签的最大特点是它是HTML唯一一个支持非矩形热区的标签，这样我们可以在图片中设定一些特殊的点击区域。
它有一个属性shape支持四种类型，与coords属性配合，规定区域的尺寸、形状和位置。

  • default，如果没设置shape时使用这个值，意味着区域覆盖整个图像。
  • circle或者circ，圆形，coords 支持三个值，分别表示中心点的 x,y 坐标和圆形半径 r。
  • rect或者rectangle，矩形，coords 支持两个值，分别表示两个对角顶点 x1，y1 和 x2，y2。
  • poly或者polygon，多边形，coords 至少包括 6 个值，表示多边形的各个顶点。

替换标签

替换标签都有一个特点，引入一个外部资源来进入页面，替换掉自身的位置。比如说我们的页面中有很多图片，或者一个短视频，都是利用替换标签的链接加载了资源后替换产生的。

script

script标签在HTML通常是用来加载JavaScript脚本的，但是也根据使用的方式不同，它不能完全定义为替换标签。
我们一般有两种加载方式：

第一种是直接把脚本代码写在标签中，第二种是通过链接引入文件，两种方法的结果是等效的，但是第二种会产生替换效果。

这里需要注意一点，替换标签都是通过src来引入文件的，而前面说的链接标签都是通过href来引入的。所以link在引入CSS的时候使用的是href。

img

img标签通过src引入一张图片替换掉所在位置。可以说img标签创建的是被引用图像的占位空间。

img标签的src属性也可以接受其它协议的uri，如http、https、file、data。
我们还可以通过width和height来设置图片的大小，可以只设置其中一项进行等比例缩放，在考虑性能方面最好是进行设置，否则了不能预知的图片尺寸后，会对之后的元素产生重排。img标签在部分浏览器中还可以通过loading来实现懒加载功能，逐步提升来性能优化。

img的alt属性也是比较重要的属性，用于用户在一些无法查看图片时的场景如：网速太慢，浏览器禁用图像，用户使用的是屏幕阅读器等，来代替图像显示在浏览器中的内容。对于视障用户来说，alt 属性通常是他们了解图像内容的唯一方式。

video

video标签用来加载视频，在一开始也是使用src进行加载，但是后来因为视频的格式不同，并且各个浏览器支持的格式也不同，video标签也提倡使用source配合加载视频。source标签除了支持media之外，还可以使用type来区分源文件的使用场景。

video标签还支持track标签，为视频添加字幕等功能，其必须使用srclang属性来指定语言，它使用kind的五个属性来定义如何使用：

  • subtitles：就是字幕了，不一定是翻译，也可能是补充性说明。默认值。
  • captions：报幕内容，可能包含演职员表等元信息，适合听障人士或者没有打开声音的人了解音频内容。
  • descriptions：视频描述信息，适合视障人士或者没有视频播放功能的终端打开视频时了解视频内容。
  • chapters：用于浏览器视频内容。
  • metadata：给代码提供的元信息，对普通用户不可见。

一个完整的 video 标签可能会包含多种 track 和多个 source，这些共同构成了一个视频播放所需的全部信息。

iframe

iframe标签的作用是生成一个文档框架嵌入当前页面中。
在一些业务需求中，我们的前端页面需要用到第三方提供的页面组件，通常就会使用iframe的方式引入，比如添加第三方提供的广告、天气预报、社交分享插件等等。
但正是因为这些内容是第三方提供的，默认情况下他们不受我们的控制，他们可以在iframe中运行JavaScirpt脚本、弹出对话框等等，破坏的用户体验和造成安全问题。
比如说第三方的页面被黑客攻破，替换掉了页面，或者域名到期，被抢注，然后利用用户浏览器中的安全漏洞下载安装木马、恶意勒索软件等等。还有点击劫持，嵌套网页等恶意操作，很可能会危害到用户的财产安全。

好在，新标准中加入了sandbox模式和srcdoc属性。
sandbox模式实现了“Secure By Default”原则，也就是说如果添加这个值为空，那么浏览器将会对iframe实施史上最严厉的调控限制，基本上除了允许显示静态资源以外，其他什么都做不了。连origin都会被强制重新分配一个唯一的值，换句话讲就是iframe中的页面访问它自己的服务器都会被算作跨域请求。当然也可以利用属性来配置，属性有：

  • allow-forms：允许iframe中提交form表单；
  • allow-popups：允许iframe中弹出新的窗口或者标签页（例如，window.open()，showModalDialog()，target=”_blank”等等）；
  • allow-scripts：允许iframe中执行JavaScript；
  • allow-same-origin：允许iframe中的网页开启同源策略。

srcdoc属性的内容是一段HTML代码，这些代码会被渲染到iframe中。如果浏览器不支持srcdoc属性，则会渲染src属性表示的内容。

对于这个标签的使用，在一些场景下一定要做好防备措施。确保安全性。

总结

经过这两节的内容，语义、元信息、链接、替换标签从分类上基本上涵盖了所有的标签，但是具体的每一个标签还是需要在标准中去看的，标签不光要关注它的使用，也应当关注标签背后的知识。
