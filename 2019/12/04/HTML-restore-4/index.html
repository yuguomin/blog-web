HTML重塑(4)——语法及扩展

Posted on 2019-12-04 | In HTML Restore   |   字数统计: 1.7k(字)   |   阅读时长: 6(分) | Visitors

前言

  • 本篇内容主要是HTML的语法学习和语言标准补充学习。

前面的内容分类学习了HTML中的标签知识，HTML虽然不是一门编程语言，而是标记语言，但它也有自己的语法特点。在了解了它的一些基础语法之后，在了解一个它的扩展应用ARIA。

HTML语法

在第一章HTML的历史与架构内容中，讲过了标记语言与编程语言的区别，并且讲到过HTML是在SGML（标准通用标记语言）的基础上创造的。所以HTML继承了一些SGML的语法特性。

标签语法

标签产生元素，我们的一个页面就是利用一个个标签的嵌套或者并列，以及他们的各种特性，展现出来的。那么如何界定它们之间的关系，或者说如何描述一个标签的位置，就是标签语法的一个主要作用。

在HTML中，描述一个元素的标签分为开始标签、结束标签和自闭合标签。开始标签和自闭合标签中可以设置标签属性：

  • 开始标签：<tagname>
    • 带属性的开始标签：<tagname attributename="attributevalue">
  • 结束标签：</tagname>
  • 自闭合标签： <tagname />
    • 带属性的自闭合标签：<tagname attributename="attributevalue" />

HTML中的开始标签名称只能使用英文字母；比较有变化性的是属性，它的属性值可以使用单引号、双引号或者不使用引号，三种情况下对应需要转义的部分不太一样：

  • 无引号属性：<tab> <LF> <FF> <SPACE> &五种字符。
  • 单引号属性：' &两种字符。
  • 双引号属性：" &两种字符。

文本语法

在HTML中，规定了两种文本语法，一种是普通的文本节点，另一种是CDATA文本节点。

普通的文本节点中，必须对<和&做转义，因为标签和文本实体中使用了这些特定的标识符，如果一段文本中包含了大量的<和&，那么可以使用CDATA，CDATA也是一种文本，它存在的意义是语法上的意义：在 CDATA 节点内，不需要考虑多数的转义情况。

注释语法

HTML 注释语法以<!--开头，以-->结尾，注释的内容非常自由，除了-->都没有问题。如果注释的内容一定要出现-->，我们可以拆成多个注释节点。

DTD 语法（文档类型定义）

DTD全称Document Type Defination，也就是文档定义模型。SGML用DTD来定义每一种文档类型，HTML属于SGML，在HTML5出现之前，HTML都是使用符合SGML规定的DTD，目的是告诉标准通用标记语言解析器要使用什么样的文档类型定义来解析文档。
在HTML4.01有三种DTD，分别是严格模式、过渡模式和 frameset 模式。
根据HTML中允许一些标签不闭合的用法，。但是，一些程序员喜欢严格遵守 XML 语法，保证标签闭合性，所以，HTML4.01 又规定了 XHTML 语法，同样有三个版本。

语言扩展ARIA

ARIA全称为Accessible Rich Internet Applications，它是可以让屏幕阅读器准确识别网页中的内容，变化，状态的技术规范。主要针对的是视觉缺陷，失聪，行动不便的人，让这类用户也能无障碍阅读。

它的核心是给HTML元素添加一个属性role，不同类型的role表示着不同元素的作用。比如说`role=”checkbox”是在告诉这个元素是一个多选框的角色：

这里就使用了列表项加入了role属性，并且加入了状态属性aria-checked，将该元素显式标识为一个复选框，并指定它在默认情况下处于选中状态。该列表项现在将添加到无障碍树中，屏幕阅读器将把它正确地报告为一个复选框。然后我们为这个元素绑定click事件，并且以checkbox的交互方式来处理用户操作。

structure角色

structure角色表示文档中的结构，自然与语义化标签有重合，建议优先使用 HTML5 标签。

window角色

window角色则代表窗体。在有的网页中，有些元素表示“新窗口”，特就是弹出的窗体。这时候，会用到 window 角色。window 系角色非常少，只有三个角色：

  • window
    • dialog
      • alertdialog
