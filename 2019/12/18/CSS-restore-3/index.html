前言

  • 本篇内容主要是CSS布局相关的学习，从传统的布局方式再到flex布局的使用，以及CSS常见的居中方式。

上一篇的内容，学习了CSS整体的语法知识，假设我们已经掌握了CSS的选择器、属性和值。然而去写页面感觉还是会很苦恼。本篇内容就是CSS功能块的布局内容，其目的就是如何把你要的盒子放在与视口相关的正确位置上。我们在拿到设计稿的那一刻，就应该开始思考如何布局了，所以掌握很多布局方式十分重要。接下来整理一些常见的布局方式。

正常流布局

正常流布局也就是我们传统盒模型布局方式，CSS在一开始的出现是为了定制HTML的样式，而早期的HTML出现的原因是让世界各地的研究报告共享，并且交叉网站。所以，早期的页面主要是以文档形式为主的，我们正常流的布局，从一开始也是基于这个方面去考虑的。

文档的特点是什么，一行行的文字，穿插着图片，引用的段落，一个返回顶部的按钮。所以正常流的布局方式就是使用display(普通流)去设置盒模型加上float(浮动)在配合上position(定位)去实现。

  1. 普通流
    一个萝卜一个坑，元素按照在html中的顺序来，块级元素(display: block)从上到下依次排列，两个元素直接的垂直距离由元素的垂直margin计算得到，是行内元素(display: inline)就和其他元素共处一行直到占满一行再换行。除另外指定，否则所有元素默认都是普通流定位。

  2. 浮动
    在浮动模型中，首先根据普通流布置盒子，然后根据浮动的方向尽可能向左或向右移动。内容可能沿着浮动的元素的一侧围绕排序(如文本环绕)。

  3. 定位(这里指绝对或者固定定位)
    元素完全脱离文档流，也就是说它与之后的兄弟姐妹元素再无瓜葛，并且按照指定的位置进行定位。

这其中涉及到块级格式化上下文、margin塌陷、浮动带来的影响等问题，具体的解决可以看我的另外一篇文章CSS-BFC。

上面的内容都是理解规则，规则很重要但是这里不多做介绍，如果有问题可以再去恶补一下。这里我们直接开始正常流布局的应用，这里主要介绍的是实现等分布局，等高布局和两列、三列自适应布局。垂直居中放在后面统一整理。

两列自适应布局

两列自适应使用场景也挺多的，比如后台系统经常左边导航右边内容，或者有的页面左边或右边广告剩下的主题。它的要求是一边固定宽或不固定，而另一边占据剩下的部分达到自适应。我们来看几种实现方法：

  1. float + margin

这种方法的处理是让左边浮动，右边margin-left左边元素宽度，达到左边固定宽右边自适应。

  1. float + overflow（BFC）

仍然按照上面的结构，这种方式是左边元素浮动，右边元素利用overflow触发BFC，不与外界元素有任何瓜葛。达到左边不固定宽右边自适应。

还有一些要求右边固定的布局，这里就不做处理的，明白原理，想一想就可以写出来。

三列自适应布局

三列自适应布局通常指的是两边宽度固定，中间一列宽度自适应，某东的PC首页就是如此。它的解决方法很多，这里先介绍几种简单的，然后介绍经典的圣杯和双飞翼布局。

流体布局和BFC

这两种方法的前提都是左右两边元素分别对应左右浮动，然后中间自适应元素利用margin挤出位置，或使用BFC来扯清关系。

这两种方法的不同是margin需要确定左右两边的宽度，而BFC不需要。但是他们也有相同的缺点，就是主要内容.center无法最先加载，当页面内容较多时会影响用户体验。

简单无脑的绝对定位方式

利用两边元素绝对定位脱离文档流到左右两边，中间元素利用margin隔离两边。

这种方法简单无脑，但是缺点在于一种场景，如果两边的内容比中间的高咋办，由于脱离了文档流，就没法撑起高度了。接下来就是圣杯布局了。

flex布局

在开始探讨flex布局之前，我们看一下之前的传统布局方式，可以看到他们想要实现两列三列包括没有说到的居中是挺麻烦的，因为传统的布局就是为了文档结构而生的，只是随着业务需要，有很多“改变盒的大小，使得它们的结构保持固定”的要求，才研究出了margin负值等这些黑科技搭配，但仍然不好理解也会有一些副作用。在这种情况下，Flex布局被随着CSS3一起提出来解决这种问题。下面很多概念内容引用自阮一峰-Flex布局。

使用一个flex布局的方法是设置元素的display属性为flex或者inline-flex，Webkit内核的浏览器，必须加上-webkit-前缀。

flex基本概念

flex布局是什么，我们一般称作“弹性布局”，任何一个容器都可以指定为Flex布局，而这个容器我们称作flex容器，它的所有子元素自动成为容器成员，称为flex项目。

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）。主轴的开始位置（与边框的交叉点）叫做main start，结束位置叫做main end；交叉轴的开始位置叫做cross start，结束位置叫做cross end。

项目默认沿主轴排列。单个项目占据的主轴空间叫做main size，占据的交叉轴空间叫做cross size。

容器的属性

在flex容器上有6个属性

  • flex-direction
  • flex-wrap
  • flex-flow
  • justify-content
  • align-items
  • align-content
  1. flex-direction属性决定主轴是哪一根以及他们的方向。

flex容器的主轴默认情况下是横轴，但是可以通过这个属性修改为纵轴，那么对应交叉轴就变为横轴。同时flex容器也有默认方向，从左到右或者从上到下，可以通过这个属性修改为从右到左或者从下往上。下面是一个有1-4的4个项目的flex容器，展示上面的效果图：

项目的属性

项目也有六个属性：

  • order
  • flex-grow
  • flex-shrink
  • flex-basis
  • flex
  • align-self
  1. order属性定义项目的排列顺序。数值越小，排列越靠前，默认为0。就好像给项目加数组索引一样。

下面的例子是text值表示文档流顺序，order值为order属性。

  1. flex-grow属性定义项目的伸缩比例。按照该比例给项目分配空间。
    这种规则是把容器的主轴等分然后按对应比例分配给项目，比如所有项目的flex-grow属性都为1，则它们将等分剩余空间。如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。默认为0，即如果存在剩余空间，也不放大。

  2. flex-shrink定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。
    这个规则在所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

  1. flex-basis属性定义了项目在不伸缩情况下的原始尺寸，主轴为横向时代表宽度，主轴为纵向时代表高度。默认值为auto，即项目的本来大小。
  1. flex属性是flex-grow,flex-shrink和flex-basis的简写，默认值为0 1 auto。后两个属性可选。即默认情况下是不放大，可缩小，默认宽度。
    该属性有两个快捷值：
  • auto (1 1 auto) 即等分，可缩小，宽度默认。
  • none (0 0 auto) 即不放大，不缩小，宽度默认。
  1. align-self属性允许单个项目有与其他项目不一样的对齐方式，可覆盖父容器align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

flex的使用属性就是这些了，通过容器属性和项目属性相互搭配，可以实现很多的功能。

接下来，我们再看一下还有一个常见的布局需求，居中。

居中

居中的要求在我们平时开发中很常见，加入有下面结构的HTML，我们想要实现的是水平垂直居中，一般有两种场景，已知元素宽高和未知宽高。

  1. table-cell + vertical-align + margin

tabel的居中方式是不太推荐的，因为vertical-align对不太了解的可能产生一些意想不到的现象，这里使用display: table-cell;转换为单元格，然后vertical-align: middle;使该行中线对齐，再利用子元素margin: 0 auto;水平居中达成效果。这里的垂直居中效果再视觉上不差，但实际是和字体挂钩有一定小偏差的。

  1. position + margin负值

子元素使用absolute和relative视场景而定，明白这两种定位的%单位相对于谁。然后根据已知宽高通过负margin修改盒子间距。

  1. position + translate

相对方法一，不实用margin而是通过CSS3的translate偏移来修正位置。这种方式由于translate百分比单位相对于自己，因此可以用在未知宽高的场景。

  1. flex

easy，通过flex容器设置元素在容器相对于轴的位置到达水平垂直居中，并且不需要知道元素具体宽高，实现多列水平垂直居中也是相当容易。

  1. position + margin: auto

这种方式，通过absolute四边定位为0，让子盒子四边margin与父容器border间距为0，再加上固定的宽度，去计算auto的margin时，平分了非自盒子border内的空间，达到了居中效果。

总结

布局对于CSS非常重要的，我们也可以发现，CSS的布局方式也是逐渐对市场需求的慢慢调整，同时好的布局方式也使我们的页面也越来越丰富。
很多场景实现的方法有很多，但是我们应该考量具体的业务环境和浏览器环境，使用符合场景需求的布局方式。
